1. Data Modeling â€“ Introduction

Introduction

Data Modeling is the process of organizing data into tables, columns, and relationships so that it can be analyzed efficiently. In Power BI, data modeling sits between data transformation (Power Query) and data visualization. A good data model ensures accurate calculations, correct filtering, and meaningful insights.

Simply put, data modeling decides how tables talk to each other.

Key Concepts

  • Data Model

  • Tables

  • Relationships

  • Fact Table

  • Dimension Table

Detailed Explanation

After data is cleaned using Power Query, it is loaded into Power BIâ€™s data model. The data model:

  • Defines how tables are connected

  • Controls how filters flow between tables

  • Supports accurate DAX calculations

A well-designed data model:

  • Improves report performance

  • Avoids ambiguity in calculations

  • Makes reports easier to build and maintain

Example Structure:

  • Sales Table (Fact Table): Sales Amount, Quantity, Date ID, Product ID

  • Product Table (Dimension): Product ID, Product Name, Category

  • Date Table (Dimension): Date, Month, Year

Summary

Data modeling is the backbone of Power BI analysis. Without a proper model, even good visuals can give wrong answers.

2. Relationship

Introduction

A relationship defines how two tables are connected through a common column. Relationships allow Power BI to combine data from multiple tables into a single report.

Key Concepts

  • Primary Key

  • Foreign Key

  • Related Tables

Detailed Explanation

A relationship is created using:

  • A column in one table (Primary Key)

  • A matching column in another table (Foreign Key)

Example:

  • Product Table â†’ ProductID

  • Sales Table â†’ ProductID

This relationship allows Power BI to:

  • Show sales by product name

  • Filter sales using product category

Without relationships, tables remain isolated and cannot work together.

Summary

Relationships connect tables and enable cross-table analysis.

3. Need of Relationship

Introduction

Relationships are essential for meaningful reporting in Power BI.

Why Relationships are Needed

  • To avoid duplicate data

  • To reduce data size

  • To ensure correct filtering

  • To support DAX calculations

Detailed Explanation

If all data is stored in one large table:

  • Data becomes repetitive

  • File size increases

  • Performance decreases

By splitting data into related tables:

  • Data becomes structured

  • Maintenance becomes easier

  • Calculations become accurate

Example Without Relationship

  • Sales report cannot display Product Category

Example With Relationship

  • Sales automatically grouped by Category

Summary

Relationships are necessary for scalability, performance, and accuracy.

4. Relationship Types / Cardinality (General)

Introduction

Cardinality describes how many rows in one table are related to rows in another table.

Key Concepts

  • Cardinality

  • One-to-One

  • One-to-Many

  • Many-to-Many

Detailed Explanation

Power BI supports three main types of relationships:

  1. One-to-One (1:1)

  2. One-to-Many (1:*)

  3. Many-to-Many (:)

Understanding cardinality is crucial because it affects:

  • Filter direction

  • Calculation results

  • Report correctness

Summary

Cardinality defines the nature of table relationships and directly impacts analysis.

5. One-to-One Relationship (1:1)

Introduction

In a One-to-One relationship, each row in Table A matches exactly one row in Table B.

Detailed Explanation

Characteristics:

  • Both tables have unique values in the linking column

  • Rarely used in real-world business scenarios

Example

  • Employee Table â†” Employee Details Table
    (Each employee has exactly one detailed record)

Summary

One-to-One relationships are simple but uncommon in analytics.

6. One-to-Many Relationship (1:*)

Introduction

One-to-Many is the most common relationship in Power BI.

Detailed Explanation

Characteristics:

  • One row in Table A relates to many rows in Table B

Example

  • Product Table (One)

  • Sales Table (Many)

One product can appear in many sales transactions.

This relationship enables:

  • Aggregation of sales by product

  • Filtering sales using product attributes

Summary

One-to-Many relationships form the foundation of most Power BI data models.

7. Many-to-Many Relationship (:)

Introduction

A Many-to-Many relationship exists when multiple rows in Table A are related to multiple rows in Table B. This type of relationship is more complex and must be handled carefully in Power BI.

Key Concepts

  • Many-to-Many Cardinality

  • Bridge (Junction) Table

  • Ambiguous Filtering

Detailed Explanation

Example Scenario:

  • A Student can enroll in multiple Courses

  • A Course can have multiple Students

If connected directly:

  • Filters may behave unpredictably

  • Calculations can give incorrect results
