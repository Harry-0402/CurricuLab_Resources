1. What is DAX?

Introduction

DAX (Data Analysis Expressions) is a formula language used in Power BI to create custom calculations for data analysis. DAX is mainly used to create Calculated Columns, Measures, and Calculated Tables.

If Power Query prepares the data, DAX analyzes the data.

Key Concepts

  • DAX

  • Formula Language

  • Calculations

  • Evaluation Context

Detailed Explanation

DAX is designed to work with:

  • Tables

  • Columns

  • Relationships in the data model

DAX formulas look similar to Excel formulas but work differently because:

  • They operate on entire columns or tables

  • They respect relationships in the data model

  • They use context (row context and filter context)

Example DAX Formula:

java
Total Sales = SUM(Sales[SalesAmount])

This formula calculates total sales across the entire dataset, respecting filters applied in reports.

Summary

DAX is the analytical engine of Power BI, enabling advanced calculations beyond simple aggregations.

2. Calculated Column

Introduction

A Calculated Column is a new column created using a DAX formula. The value is calculated row by row and stored in the data model.

Key Concepts

  • Row Context

  • Stored Values

  • Column-Level Calculation

Detailed Explanation

Calculated Columns:

  • Are evaluated for each row

  • Increase model size (stored in memory)

  • Are useful for categorization and labeling

Example:

ini
Profit = Sales[SalesAmount] - Sales[Cost]

This creates a Profit value for each sales transaction.

Use Cases

  • Creating profit per row

  • Creating year or month columns

  • Creating conditional labels

Summary

Calculated Columns are best suited for row-level calculations and data enrichment.

3. Measures

Introduction

Measures are DAX calculations that are evaluated at query time, not stored in the data model.

Key Concepts

  • Filter Context

  • Dynamic Calculation

  • Aggregation

Detailed Explanation

Measures:

  • Are calculated dynamically based on filters

  • Do not increase data size

  • Change values depending on visuals

Example:

java
Total Profit = SUM(Sales[Profit])

If a report is filtered by Year or Region, the measure recalculates automatically.

Common Use Cases

  • Total Sales

  • Average Sales

  • Growth Percentage

  • KPIs

Summary

Measures are the preferred way to perform aggregations and analytical calculations in Power BI.

4. Calculated Columns vs Measures

Comparison Table

FeatureCalculated ColumnMeasure
EvaluationRow by rowOn demand
StorageStored in modelNot stored
ContextRow contextFilter context
Use CaseCategorizationAggregation

Explanation

  • Use Calculated Columns when you need row-level data

  • Use Measures for totals, averages, and KPIs

Summary

Choosing the correct calculation type improves performance and accuracy.

5. DAX Table and Column Name Syntax

Introduction

DAX uses a specific syntax to reference tables and columns.

Syntax Rules

  • Table names are enclosed in single quotes if they contain spaces

  • Column names are always inside square brackets

Examples

sql
Sales[SalesAmount]
'Sales Data'[Order Date]

Summary

Correct syntax ensures error-free DAX formulas.

6. Creating Calculated Columns

Steps

  1. Go to Data View

  2. Select table

  3. Click New Column

  4. Write DAX formula

  5. Press Enter

Example

java
Sales Category =
IF(Sales[SalesAmount] > 50000, "High", "Low")

Summary

Calculated Columns are created directly inside the table for row-level logic.

7. Creating Measures

Steps

  1. Go to Report View or Data View

  2. Select table

  3. Click New Measure

  4. Write DAX formula

Example

java
Total Sales = SUM(Sales[SalesAmount])

Summary

Measures provide flexible and dynamic analytics.
