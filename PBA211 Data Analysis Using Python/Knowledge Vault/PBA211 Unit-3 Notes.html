1. Broadcasting in NumPy

Introduction

Broadcasting is a powerful feature in NumPy that allows operations between arrays of different shapes.

In simple words:

Broadcasting allows NumPy to automatically adjust array sizes so that mathematical operations can be performed.

Without broadcasting, we would need loops. With broadcasting, NumPy handles everything internally and efficiently.

Key Concepts

  • Broadcasting

  • Shape compatibility

  • Implicit expansion

  • Element-wise operations

  • Axis alignment

2. Why Broadcasting is Needed?

Let us understand with a simple example.

Example 1: Adding a scalar

python
a = np.array([1,2,3])
a + 10

Output:

csharp
[11 12 13]

Here, 10 is automatically added to every element.

This is broadcasting.

NumPy treats 10 as:

csharp
[10 10 10]

But it does NOT actually create this array in memory. It efficiently performs the operation.

3. Broadcasting Rules

For two arrays to be broadcasted:

Rule 1:

If dimensions are not equal, NumPy adds 1 to the smaller shape.

Rule 2:

Two dimensions are compatible if:

  • They are equal

  • OR one of them is 1

4. Broadcasting Example (1D + 1D)

python
a = np.array([1,2,3])
b = np.array([4,5,6])

a + b

Shapes:

  • a â†’ (3,)

  • b â†’ (3,)

Since shapes match â†’ Direct addition.

5. Broadcasting Example (2D + 1D)

python
a = np.array([[1,2,3],
              [4,5,6]])

b = np.array([10,20,30])

a + b

Shapes:

  • a â†’ (2,3)

  • b â†’ (3,)

How does NumPy handle this?

Step-by-step:

lua

 [4,5,6]]

It treats b as:

lua
[[10,20,30]
 [10,20,30]]

Result:

lua
[[11,22,33]
 [14,25,36]]

This is automatic broadcasting.
